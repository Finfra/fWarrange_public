# 자주 묻는 질문 (FAQ)

## 일반

### Q: fWarrange는 무료인가요?
A: 네, fWarrange는 MIT 라이선스로 제공되는 오픈소스 프로젝트입니다.

### Q: 어떤 macOS 버전에서 동작하나요?
A: macOS 15.0 (Sequoia) 이상에서 동작합니다.

### Q: Apple Silicon(M1/M2/M3/M4)을 지원하나요?
A: 네, 네이티브로 지원합니다.

## 권한

### Q: "손쉬운 사용 권한이 필요합니다" 오류가 나타납니다.
A: 시스템 설정 > 개인정보 보호 및 보안 > 손쉬운 사용에서 fWarrange 앱(또는 터미널 앱)을 추가해야 합니다. 자세한 내용은 [설치 가이드](02_Install.md#3-손쉬운-사용accessibility-권한-설정)를 참조하세요.

### Q: 빌드할 때마다 권한이 풀립니다.
A: Xcode에서 새로 빌드하면 바이너리 서명이 달라지므로 macOS가 새로운 앱으로 인식합니다. 개발 중에는 매번 재등록이 필요합니다. 배포 빌드에서는 고정 서명으로 해결됩니다.

### Q: `tccutil reset Accessibility`를 실행하면 어떻게 되나요?
A: 모든 앱의 손쉬운 사용 권한이 초기화됩니다. 이후 필요한 앱들을 다시 등록해야 합니다.

## 레이아웃 저장/복원

### Q: 창 복원이 실패합니다.
A: 다음을 확인하세요:
1. 손쉬운 사용 권한이 부여되었는지
2. 복원하려는 앱이 실행 중인지
3. `-v` 옵션으로 매칭 점수를 확인하여 어떤 창이 실패하는지 파악

### Q: 복원 시 창 위치가 약간 어긋납니다.
A: 복원 검증 시 3px 오차를 허용합니다. 일부 앱(특히 Electron 기반)은 시스템 제약으로 정확한 위치 지정이 안될 수 있습니다.

### Q: 다중 모니터에서 음수 좌표가 표시됩니다.
A: 정상입니다. macOS 좌표계에서 메인 모니터 왼쪽에 배치된 보조 모니터의 좌표는 음수입니다. 저장 및 복원 시 올바르게 처리됩니다.

### Q: 디스플레이 배치를 변경하면 어떻게 되나요?
A: 시스템 설정에서 모니터 배치를 변경하면 이전에 저장한 좌표가 엉뚱한 위치를 가리킬 수 있습니다. 배치 변경 후 레이아웃을 새로 저장하는 것을 권장합니다.

### Q: 특정 앱의 창만 저장/복원할 수 있나요?
A: 네, CLI에서 `--app=Safari,iTerm2` 옵션을, API에서 `filterApps` 필드를 사용하면 됩니다.

### Q: 전체 화면(Full Screen) 앱도 복원되나요?
A: 전체 화면 모드의 앱은 별도의 Space에서 실행되므로 일반적인 좌표 기반 복원이 적용되지 않습니다.

## REST API

### Q: API 서버가 기본으로 비활성인 이유는?
A: 보안상의 이유입니다. HTTP 서버를 열면 로컬 네트워크 공격에 노출될 수 있으므로, 사용자가 필요할 때만 수동으로 활성화하도록 설계되었습니다.

### Q: 외부 네트워크에서 접속할 수 있나요?
A: 설정 > API 탭에서 외부 접속을 활성화하고 CIDR 화이트리스트를 설정하면 LAN 내에서 접근 가능합니다. 인터넷에 직접 노출하는 것은 권장하지 않습니다.

### Q: 포트를 변경할 수 있나요?
A: 네, 설정 > API 탭에서 포트를 변경할 수 있습니다. 기본값은 3016입니다.

### Q: Apple Shortcuts에서 호출하려면?
A: Shortcuts 앱에서 "URL 내용 가져오기" 액션을 사용하여 `http://localhost:3016/api/v1/layouts/myLayout/restore`와 같은 URL을 POST 방식으로 호출하면 됩니다. 자세한 내용은 [API 사용법](06_API_Usage.md#apple-shortcuts-연동)을 참조하세요.

## Skill / MCP

### Q: Skill과 MCP의 차이는 무엇인가요?
A:
- **Skill**: Claude Code에서 `/fwarrange:fwarrange` 슬래시 커맨드로 호출. 내부적으로 curl 명령어 실행.
- **MCP**: Claude Desktop/Code에서 AI가 자동으로 적절한 도구를 선택하여 호출. 완전 자연어 지원.

### Q: Skill과 MCP를 동시에 사용할 수 있나요?
A: 네, 독립적으로 동작하므로 동시 사용이 가능합니다. Skill은 명시적 커맨드, MCP는 자연어 기반입니다.

### Q: MCP 서버가 연결되지 않습니다.
A: 다음을 확인하세요:
1. fWarrange 앱이 실행 중이고 REST API가 활성화되어 있는지
2. `claude_desktop_config.json`의 JSON 문법이 올바른지
3. Node.js 18 이상이 설치되어 있는지
4. `npx fwarrange-mcp`를 직접 실행하여 오류 메시지 확인

### Q: fWarrange 앱 없이 Skill/MCP만 사용할 수 있나요?
A: 아닙니다. Skill과 MCP 모두 fWarrange 앱의 REST API 서버를 통해 동작합니다. 앱이 실행 중이어야 합니다.

## 성능

### Q: 창이 많으면 복원이 느린가요?
A: 기본적으로 앱별 병렬 복원(`enableParallel: true`)이 활성화되어 있어, 20-30개 창도 수 초 내에 복원됩니다.

### Q: YAML 파일이 너무 커지면?
A: 일반적인 사용에서 YAML 파일은 수십 KB 이하입니다. 불필요한 레이아웃은 정기적으로 삭제하는 것을 권장합니다.
