# fWarrange Layout Management Workflow

**Description**:
사용자가 fWarrange 창 레이아웃 캡처, 복원, 목록 조회 등 실행 중인 앱과 관련된 작업을 요청할 때 사용하는 워크플로우입니다.

**Trigger**:
사용자가 "현재 창 상태 저장해줘", "레이아웃 복원해줘", "저장된 레이아웃 목록 보여줘" 등 fWarrange 앱의 기능을 요청할 때.

**Steps**:
1. **API 상태 확인**: `fwarrange-api` 스킬의 Health Check(`curl http://localhost:3016/`)를 실행하여 앱의 REST API가 정상적으로 응답하는지 확인합니다.
2. **오류 처리**: 응답이 없거나 접속이 거부될 경우, 사용자에게 "fWarrange 앱이 실행 중인지, 설정에서 API 서버 기능이 켜져 있는지 확인해 주세요."라고 안내하고 작업을 중단합니다.
3. **작업 수행**: API가 정상 동작하면, 사용자의 요청에 맞는 `fwarrange-api` 스킬의 cURL 명령을 실행합니다 (예: 캡처, 복원, 목록, 삭제).
4. **결과 보고**: API 응답 JSON 결과를 해석하여, 어떤 작업이 수행되었는지 (예: 캡처된 창의 개수, 복원 성공/실패 여부 등) 사용자에게 간결하고 명확하게 한국어로 보고합니다.
